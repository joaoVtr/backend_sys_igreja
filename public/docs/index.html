Introduction

Project to cadastre churches and members, and input member in a church

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

http://localhost/

Authenticating requests

This API is not authenticated.

GET api/churches

Store a new church.

Example request:
curl --request POST \
    "http://localhost/api/churches" \
    --header "Content-Type: multipart/form-data" \
    --header "Accept: application/json" \
    --form "name=fugiat" \
    --form "address=architecto" \
    --form "website=sed" \
    --form "picture=@/tmp/phpBD9QYr" 
const url = new URL(
    "http://localhost/api/churches"
);

const headers = {
    "Content-Type": "multipart/form-data",
    "Accept": "application/json",
};

const body = new FormData();
body.append('name', 'fugiat');
body.append('address', 'architecto');
body.append('website', 'sed');
body.append('picture', document.querySelector('input[name="picture"]').files[0]);

fetch(url, {
    method: "POST",
    headers,
    body,
}).then(response => response.json());

URL Parameters

Update one church.

Example request:
curl --request PUT \
    "http://localhost/api/churches/1" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"name\": \"Igreja N2\",
    \"address\": \"non\",
    \"website\": \"expedita\"
}"

const url = new URL(
    "http://localhost/api/churches/1"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "name": "Igreja N2",
    "address": "non",
    "website": "expedita"
};

    method: "PUT",
    headers,
